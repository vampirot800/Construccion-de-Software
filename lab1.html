Preguntas a responder

¿Cuál es la diferencia entre Internet y la World Wide Web?
Internet es la infraestructura de red global mientras que el WWW es un servicio que permite el acceso a recursos en internet
¿Cuáles son las partes de un URL?
Protocolo, Nombre de dominio, Subdominio, Ruta...
¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
GET: solicitar datos de un recurso, HEAD: solicitar solo los encabezados de respuesta HTTP, POST: enviar datos al servidor para ser procesados, PUT: enviar datos al servidor para ser almacenados en una ubicación específica, PATCH: realizar modificaciones parciales a un recurso existente, DELETE: solicitar la eliminación de un recurso específico en el servidor
¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
POST, ya que es más adecuado para enviar formularios que contienen datos sensibles porque proporciona una mayor seguridad y protección de la información confidencial
¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
GET, se usa para solicitar datos de un recurso específico, como una página web, y es el método predeterminado que utilizan los navegadores cuando introduces una URL
Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
El código de estado HTTP 200 indica "OK", no es un error, ya que la solicitud se procesó correctamente
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
Sí, el error 404 indica que el servidor no pudo encontrar el recurso solicitado, lo que puede deberse a varias razones, como un enlace roto, un archivo eliminado o movido, una URL mal escrita, entre otros.
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
Sí, el error 500 indica un error interno del servidor, lo que significa que algo salió mal en el servidor al procesar la solicitud del usuario.
